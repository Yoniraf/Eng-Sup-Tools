Getting started (new users)

Open the toolbox in index.html, pick a tool from the sidebar, and use the top bar to search and share deep links. Many tools store state in browser localStorage for convenience — use “Clear” buttons when working with sensitive data.

Basics

  • Open the toolbox: hosted link or local server.
  • Deep links: share index.html?tool=<tool-id>.
  • Open in new tab: use the top bar action to work full-screen.

SyncApp tools & browser limitations

  • Some SyncApp endpoints return Cross-Origin-Resource-Policy: same-origin, so hosted browser calls can fail with TypeError: Failed to fetch.
  • Recommended: use SyncApp ERP API Runner (Desktop Download) (starts a proxy automatically).
  • Alternative: use the web runner + an HTTPS proxy (HTTPS page + HTTP proxy will be blocked as mixed content).

JSON Studio

A multi-purpose JSON workspace: side-by-side editors, JSON diff, CSV import/export for arrays-of-objects, Coralogix log reconstruction, and a nested key-value extractor.

How to use

  • Choose a mode: each side supports Tree, Text, Table.
  • Compare: click Compare to see a “parsed JSON diff” output.
  • Beautify: click Beautify both to pretty-print JSON on both sides (only if parseable).
  • Tabs: use + to add tabs; double-click a tab name to rename; close (×) when multiple exist.

Inputs & outputs

  • Diff output: line-by-line path differences like $.a.b: 1 ⇢ 2.
  • CSV import: imports a CSV file into an editor as JSON and switches that side to Table mode.
  • CSV export: only works when the JSON is an array of objects (e.g. [{...},{...}]).
  • Persistence: tabs and editor state are saved in browser localStorage.
Coralogix Log Reconstructor: upload a log export (JSON array, single JSON object, or NDJSON). The tool sorts by LogPart, concatenates message parts (removing a leading [n] prefix), then tries to parse JSON (best-effort fixes for missing closing braces/brackets). It downloads either reconstructed_message.json (valid JSON) or reconstructed_message_raw.txt (raw). You can also inject the reconstructed output into the left/right editor.
Nested JSON Key-Value Extractor: paste a JSON array, refresh keys, select dot-path keys (e.g. a.b), then extract into a JSON array of objects. You can load from an editor or inject extracted JSON back into an editor.

Field Mapping Default Values Wizard

A step-by-step wizard to generate a FieldMappingDefaultValues JSON payload from a requestor CSV + DB results, and generate a ready-to-run curl command (useful when the browser is blocked by mixed-content/CORS).

How to use

  • Environment + instanceId: select Sandbox/Production and enter instanceId.
  • Requestor CSV: upload or paste CSV with required columns: defaultValueType, value ID, value name, applyToPayment, isReadOnly, Entity name.
  • PayerEntityMapping: run the tool’s query, paste results (including headers), click Parse, and confirm the right columns.
  • FieldsMapping: run the tool’s query to find fieldMappingId and enter it.
  • Generate: click Generate JSON, then copy/download.
  • Call API: click Copy curl, paste into terminal, replace <PASTE_BEARER_TOKEN>.

Outputs & notes

  • Payload: array of objects containing fieldMappingId, defaultValueType, externalId, externalName, applyToPayment, isReadOnly, payerEntityMappingId.
  • Merge behavior: if an “Entity name” from the requestor CSV doesn’t match the DB paste, it becomes a merge error and is skipped.
  • Security: the tool doesn’t store secrets; still clear tokens/inputs after use.

String Tools • Line Breaks

Converts lists between line breaks / commas / spaces / custom delimiter, with optional trimming, de-duplication, wrapping, and a SQL helper that builds an IN (...) clause (with proper single-quote escaping).

How to use

  • Paste values (one per line, or comma/space separated).
  • Select conversion (e.g. line breaks → comma).
  • Optional: Trim, Remove duplicates, Wrap each item with before/after strings.
  • Optional: enable SQL helper to output IN ('a','b') (auto-quotes and escapes single quotes).
  • Click Execute, then Copy to clipboard.

Notable behavior

  • Item detection: if the input contains newlines, it splits by newline; else if it contains commas, it splits by comma; else splits by whitespace.
  • Comma output formatting: can be “one per line” and can add a trailing comma.
  • SQL quoting: values are wrapped with single quotes; internal ' becomes ''.

TSV/CSV → JSON Converter

Converts tab-delimited text (TSV) into a JSON array of objects. Despite the name, the delimiter is fixed to \t.

How to use

  • Paste data where the first line is headers (tab-separated).
  • Click Convert to generate JSON.
  • Click Copy to copy the JSON output.
  • Click Clear to reset input/output.

Output shape

  • Produces [{header1:value1, header2:value2, ...}, ...].
  • Missing cells become empty strings ("").

SQL Template Generator

Generates one SQL statement per CSV row by replacing {placeholder} tokens with values from CSV headers.

How to use

  • Write/paste a SQL template containing placeholders like {tipaltiid} or {externalid}.
  • Upload a CSV file.
  • Click Generate Queries.
  • Click Copy to Clipboard to copy the generated output.

Important notes

  • CSV parsing is simple: it splits by newline and then by comma (no quote-aware CSV parsing).
  • Replacement is global per header: all occurrences of {HeaderName} are replaced.
  • Types: replacement is plain text; you must include quotes in your template if needed.

Retry Specific SyncEngine (Batch Retry)

Calls SyncEngine’s scheduleRetrySync endpoint in batches, with retries, delays, a special “active sync job” wait path, and verbose logs.

What you need

  • Environment: Production or Sandbox (selects endpoint URL).
  • Authorization bearer token: used in the Authorization: Bearer … request header.
  • recordType: e.g. Vendor.
  • ID Type: externalId or tipaltiId (auto-sets direction).
  • IDs: paste values, or upload CSV (the tool tries to find an ID-like column; else uses the first column).

How it behaves

  • Batching: splits IDs into batches of “IDs per batch”.
  • Retries: on failures, retries up to “Max Retries per Batch” with a delay.
  • Special case: if response includes “instance has an active sync job”, it waits 1 minute (with an option to “Manual Retry (Skip Wait)”).
  • Logs: prints the request payload per batch and response status into the Results panel.
Security note: bearer tokens are sensitive. Prefer short-lived tokens, avoid screen sharing, and clear the tool input when done.

SyncApp ERP API Runner

A Postman-style runner for SyncApp ERP APIs (NetSuite / Intacct / QBO): single-run or runner mode (CSV/paste), with aggregated JSON output you can copy into tickets/tools.

What you need

  • Environment: sbox or production.
  • ERP: NetSuite / Intacct / QBO.
  • Bearer token (only remembered if you opt in).
  • Action: choose from the built-in registry and fill placeholders (e.g. payerPublicId, id, recordType).

Proxy + runner mode

  • Proxy: if you see TypeError: Failed to fetch / status 0, enable Use local proxy and set a proxy base URL. If this page is HTTPS, the proxy must be HTTPS too.
  • Runner mode: iterate a single variable list (paste) or provide full CSV rows to override multiple variables per request.
  • Output: aggregated JSON with meta + results[], including per-iteration resolved vars, URL, status, elapsed time, and parsed body (when possible).
Security note: avoid sharing bearer tokens in tickets or screen shares; use the Clear buttons when done.

SyncApp ERP API Runner (Desktop Download)

A Windows/macOS desktop wrapper for the SyncApp ERP API runner UI. Recommended because it avoids browser CORP/CORS issues by automatically starting a local proxy and pre-configuring the web runner to use it.

Why desktop

  • Some SyncApp hosts respond with Cross-Origin-Resource-Policy: same-origin, which breaks browser fetch from hosted tools.
  • The desktop app behaves like the old runner executable: it just works (proxy is automatic).

Download & use

  • Download: open the latest release and pick the right installer for your OS.
  • Run: open the app, paste bearer token, choose ERP/action, run once or runner mode.
Latest downloads: https://github.com/yoniraf/Eng-Sup-Tools/releases/latest

Update SRS Generator

Generates a JSON array payload for a list of IDs (one per line): [{ id, direction: 2, recordType: "bill", updateSRS: true }, ...].

How to use

  • Paste IDs (one per line).
  • Click Generate.
  • Click Copy to Clipboard.

Notes

  • Empty lines are ignored; IDs are trimmed.
  • Output is pretty-printed JSON (2-space indentation).

Bulk Update Record State (MongoDB bulkWrite Generator)

Generates a MongoDB script for db.getCollection("syncengineservice_recordsyncstates").bulkWrite([...]) containing updateOne operations that set a mapping field and also sets hasBeenSynced: true.

Inputs

  • Instance ID and Record Type (used in every filter).
  • Pairs (one per line): supports tipaltiId,externalId, space-separated pairs, JSON per line, or CSV with a header.
  • Update direction:
    • Filter by tipaltiId → update externalId, or
    • Filter by externalId → update tipaltiId.
  • Optional: de-duplicate by the filter field; optional “ordered: false” for bulkWrite.

Output

  • A ready-to-run MongoDB script with a bulkWrite call.
  • KPIs: number of pairs, ops, and validation errors.
  • Buttons to copy output or download as a .js file.
Important: This tool does not execute anything—it only generates a script. Review filters/updates before running in any environment.

SQL CASE WHEN Generator

When you need results returned in the same order as items in an IN (...) list, this appends an ORDER BY CASE clause.

How to use

  • Paste a query containing IN (...).
  • Click Generate CASE WHEN Query.
  • Copy the “Generated Query” output.

Important notes

  • It searches for the first IN(...) using a regex; it does not parse full SQL.
  • The generated clause is hard-coded to ORDER BY CASE idAtPayer (edit the column name if needed).
  • Duplicates are removed while keeping the first-seen order; unknown values fall back to ELSE 999.

API Key / Hash Generator

Generates an HMAC-SHA256 hash (Base64) from the concatenation payerName + idap + timestamp + eat using a secret key.

How to use

  • Fill payerName, idap, timestamp and key (eat is optional).
  • Click Generate near timestamp to fill current milliseconds.
  • Click Generate hash then Copy hash.

Output

  • Hash is Base64-encoded HMAC-SHA256.
  • Copy actions use your clipboard; a browser alert confirms copy.
Security note: treat key as secret. Clear fields after use.

CSV Splitter

Splits a CSV into multiple smaller CSV files. Each output file includes the original header row plus a chunk of data rows.

How to use

  • Drag & drop a .csv file (or click the dropzone).
  • Set “Lines per file” (counts data rows; header is always included).
  • Click Split CSV.

What happens

  • Your browser downloads multiple files named original_part1.csv, original_part2.csv, …
  • CSV parsing is line-based; empty lines are ignored.
  • All work is local in the browser.
